## 試験の説明 {#exam-description}

{{exam-description}}

<br>

### 試験の詳細 {#exam-details}

* レベル：エキスパート（1～3 年の経験）
* 合格点：45/68
* 時間：136 分
* 配信：オンライン検索（カメラへのアクセスが必要）
* Language: English
* 価格：$225 （グローバル）/$150 （インド）
* 試験 ID: AD0-E605

<br>

### 対象読者 {#intended-audience}

* ソフトウェア開発者
* データエンジニア
* データアーキテクト
* ソリューションアーキテクト
* エンタープライズアーキテクト

<br>

### 最低限の経験 {#minimum-experience}

以下が必要です。

* セグメント評価の違い（ストリーム、バッチ、エッジ）の理解
* API のユースケース（デベロッパープロジェクトの作成方法、認証の仕組み）について
* ID サービスについて
* データフローの理解
* Adobe I/Oの理解
* 関係の仕組みについて
* 顧客のユースケースのデータモデル（スキーマ、ID、関係）を作成する機能
* 結合ポリシーに基づいてデータを結合する戦略を立てる機能
* データのラベル付け、ポリシーの適用、Privacy Serviceに関する正しい方法を判断する方法の理解
* データ取り込みパターンの制限に精通していること
* 宛先と使用する宛先について
* エッジデータ収集フレームワークについて
* 設定する必要がある AEP 権限（属性レベルの権限）を決定する機能（クライアントにレコメンデーションを行う）
* クエリサービスのユースケースの基本的な理解
* Sources/destinations SDK の基本的な理解

次の概念に精通している必要があります。

* エンティティ関係図
* 管理者/権限
* Experience platform API
* Adobe Experience Platform デバッガー
* Web およびモバイル SDK

以下の環境を快適に使用できます。

* AEP
* エッジネットワーク

<br>

### 試験の目的と範囲 {#exam-objectives}

{{exam-objectives}}

**データ・アーキテクチャ（19%）**

* シナリオに基づいて、RDBMS をAdobeに翻訳します [!DNL RT-CDP]の NoSQL データモデル
* リアルタイム顧客プロファイルモデリングのベストプラクティスの示し
* モデリングのための ID 戦略と関係の設計

**リアルタイム顧客プロファイル （15%）**

* プロファイルアセンブリの仕組みの説明
* ID グラフの仕組みの説明
* プロファイル構成の説明
* エッジプロファイルとハブプロファイルの区別

**データ取り込み（13%）**

* CDP を使用したデータ取得機能について
* Edge 取り込み機能の説明
* データ取り込みアーキテクチャに関する高度な手法の説明

**セグメント化（18%）**

* CDP 内でオーディエンスとセグメントを作成する様々な方法の説明
* 様々なセグメント化のタイプとその動作について説明する
* セグメント/オーディエンスアクティベーションにユースケースを適用する方法の理解を示す

**アクティベート （12%）**

* CDP 内の様々なアクティベーションパターンの説明
* アクティベーションに関するガードレールの説明
* サイトのパーソナライゼーションについての説明

**ガバナンス（9%）**

* ガバナンスのベストプラクティスをAdobeに適用 [!DNL RT-CDP] 実装
* 様々なサービスをまたいだデータの管理方法について説明します

**管理（15%）**

* 監視機能について
* アラートの説明
* 異なるライセンス使用状況の説明
* Adobe内での ABAC 有効化のユースケースの説明 [!DNL RT-CDP]
