# 資格認定のAdobe[!DNL Workfront] 更新

Adobe認定は 2 年間有効です。 この 2 年に近づいている場合は、証明書を更新してアクティブな状態に保ちます。

まず、下のタブ（プロフェッショナル、エキスパート、マスター）で適切なレベルを選択します。 その後、資格認定を更新するために必要なことを慎重に確認します。

資格認定の有効期限が切れる前に、すべての要件を完了するための十分な時間を確保してください。

資格認定の有効期限が切れると、資格認定テストを再受験する必要があることに注意してください。これは無料ではありません。

### 準備する

**試験の詳細：**

* 合格点：26/34
* 時間：68 分
* 料金：無料
* 配信：オンデマンド/順不同
* 対象言語：英語
* 前提条件：現在の（期限切れでない）Adobe[!DNL Workfront]Fusion Developer Professional 認定

**範囲と目的**

セクション 1: コアシステムの管理とセットアップ（26%）

* Fusion シナリオで、別の関数内にネストする関数が必要な場合は、適切に形成された式を選択します。
* タイムゾーンの操作が必要な Fusion シナリオの場合、ユーザー、表示可能な組織、タイムゾーンの上書きが必要な場所を区別します。
* Fusion シナリオを指定した場合、入力データを出力データに適切に変換するマッピングパネル式やモジュールを選択します。
* Fusion 開発ツールを使用して、実行エラーのトラブルシューティングや、サードパーティシステムに対する呼び出しと応答を特定する正しい方法を特定します。

セクション 2：シナリオの設計とアーキテクチャ（47%）

* 手順のリストを指定した場合は、JSON を解析し、バンドルに変換する正しいシーケンスを選択します。
* Fusion のシナリオを前提として、設計を簡略化したり、メンテナンスのために最適化したりする機会を特定します。
* 状況を考慮して、シナリオでのデータフローを減らす機会を特定します。
* 検索モジュールに表示されるバンドル数の例と次のアグリゲータを使用して、バンドルインスペクターで処理および表示される操作数の変化を説明します。
* エラー 403:forbidden の例を考えると、エラーメッセージの生成元と原因を正しく選択します
* 検索および読み取りモジュールでの参照およびコレクション [!DNL Workfront] 適切な使用方法を特定します。

セクション 3: テストとエラー処理（15%）

* Fusion のシナリオを考慮して、信頼性の低いサービスを防止または処理するためのソリューションを正しく特定します。
* Fusion シナリオに必要なデータが欠落している場合は、無効なデータの処理方法を選択します。

セクション 4:API の使用（12%）

* 専用アプリを持たないサードパーティシステムの場合は、HTTP アプリを特定し、適切なモジュールを選択します。
* CRUD 操作と、REST API のその他の一般的な機能との違いを特定します。

* プロジェクトを管理する様々な方法を特定します。 （ウォーターフォール、アジャイル、スクラム、かんばんを検討してください。）
* シナリオを前提として、関係者に必要な情報を提供する方法を説明します。 （レポートのエクスポート、ファイルのエクスポート、繰り返し配信などの検討）
* シナリオを想定して、日付タイプと期間がプロジェクトの条件に与える影響を決定します。 （予定開始、見込み開始、個人のコミット日などを考慮）
* シナリオを使用して、システムレベルまたはグループレベルのカスタムステータスの変更を実装または変更する方法を決定します
* シナリオを考えた場合、レポートを作成する手順と、必要なレポートのタイプを決定します。
* シナリオを前提として、イシューをプロジェクトで利用するタイミングを決定します。

セクション 3: クローズ アウト （12%）

* 特定のオブジェクトステータスを指定した場合は、実行する必要のあるアクティビティやトリガーされるアクションを決定します。 （承認待ちを考慮します。つまり、承認が必須であるか、承認が解決されるまでプロジェクトをクローズできない、アクティブなプロジェクトでのみ通知が送信される、完了したプロジェクトの計画、データ編集、ログ時間のあるタスクの削除に相当するステータスではない、など）。
* プロジェクトの成功の測定に使用できるプロジェクト要素を特定します。 （ベースラインのバリアント、財務、予定日と実際の日付/時間/コスト、カスタムデータ/KPI などを考慮します）。 ケース固有の質問は使用しないでください。 技術的な要素に焦点を当てます。）
