### 対象読者 {#intended-audience}

* データアーキテクト/エンジニア

### 最低限の経験 {#minimum-experience}

以下が必要です。

* ～の基礎知識 [!DNL Experience Platform] UI 内を移動できる
* セグメントを作成し、オーディエンスを作成する機能（ロジックが含まれています）
* 宛先を設定する機能
* セグメントを宛先に接続する機能
* リアルタイムで応答および反応するインサイト/イベントを作成する機能
* データ取り込み時の DULE 設定、アクティベーション時のセグメント制限について
* チャネル別のオプトイン要件の理解
* ユースケースのデータの頻度と量に関するベストプラクティスを使用したときに取得およびアクティブ化する必要があるデータを設計/理解する機能
* 様々なチャネルで必要なデータを理解する機能
* 必要なプロファイル属性を決定する機能
* 宛先からセグメントを送信する頻度に関する知識
* セグメントメンバーシップの更新頻度に関する知識

## 要件#1:Adobeに関する情報 [!DNL Experience Platform] 技術基盤（AD0-E600） {#req-1}

AD0-E600 試験は、基本的な機能やAdobeで頻繁に実行される作業をカバーするスキルと知識を検証します [!DNL Experience Platform] エキスパートレベルで。

### AD0-E600 の試験の詳細 {#exam-details-600}

* レベル：エキスパート（1～3 年の経験）
* 合格点：35/50
* 時間：105 分
* 配信：オンラインで調達（カメラへのアクセスが必要）またはテストセンターで調達
* Language: English
* 価格：$225 （グローバル）/$150 （インド）
* 試験 ID: AD0-E600

<br>

### AD0-E600 の試験の目的と範囲 {#exam-objectives}

{{exam-objectives}}

**セクション 1：データモデリング（20%）**

* ソースデータを分析して、プロファイルステッチのプライマリ ID とセカンダリ ID を評価します。
* UI を使用して XDM スキーマを作成/編集する方法の理解を示します。
* DULE ガイダンスの特定

**セクション 2：データ取り込み（36%）**

* 取り込み用のデータのフォーマットおよび準備方法を示す
* OOTB コネクタを使用してデータソースを接続する方法を示す
* バッチとストリーミングを使用してソースデータを取り込む方法を示します
* データ転送の監視方法
* ソースデータに対してデータ検出を実行する方法を示す
* XDM に一致するようにデータを変換する方法を示す

**セクション 3：プロファイルの統合（34%）**

* 取り込み後のプロファイルとイベントデータの検証
* ID 名前空間の定義
* プロファイルサービスでの ID グラフの使用方法の説明
* Data science workspace モデリングサービスを使用してプロファイルをエンリッチメントする方法を特定します
* セグメントビルダーを使用したセグメントの作成方法とセグメント化サービスの仕組みを示します。

**セクション 4：有効化（10%）**

* 宛先の設定方法とセグメントのアクティベーションの仕組みを示します
* データアクセス API とReal-Time CDPを使用したデータの書き出しのデモと理解

## 要件#2:Adobeに関する情報 [!DNL Real-Time CDP] 技術担当者（AD7-E601） {#req-2}

<br>

### AD7-E601 の試験詳細

* レベル：エキスパート（1～3 年の経験）
* 合格点：25/34
* 時間：51 分
* 配信：オンラインで調達（カメラへのアクセスが必要）またはテストセンターで調達
* Language: English
* 価格：$225 （グローバル）/$150 （インド）
* 試験 ID: AD7-E601

<br>

### AD7-E601 の試験の目的と範囲

{{exam-objectives}}

セクション 1：検証（41%）

* プロファイルが適切な属性とセグメントを持っていることを検証するために必要な適用概念
* 機能要件または技術要件に基づいて導入戦略を検証する方法の理解を示す
* RTCDP のエンドツーエンド（E2E）フローを実装するための関連変数の分析
* 宛先へのセグメント書き出しの監視
* データガバナンスポリシーの検証に必要な概念の適用

セクション 2：セグメント化（32%）

* ビジネス要件に基づいたセグメントの作成方法の理解を示します
* 取り込んだデータに基づいてセグメント評価を決定する方法の理解を示す

セクション 3：宛先（27%）

* ビジネスユースケースに基づいてオーディエンスを送信/書き出す際に、1 つ以上の宛先を設定するために必要な概念を適用します
* Destination を通じたセグメントの書き出しに関する理解を示す
